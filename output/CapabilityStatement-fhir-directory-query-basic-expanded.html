CapabilityStatementFhirDirectoryQueryBasicExpanded

Capabilities for a Basic Federated Query Server for where expanded search capabilities are met

ModeSERVER
Description

A Federated Query server SHALL:

  1. Support all profiles defined in this Implementation Guide..
  2. Implement the RESTful behavior according to the FHIR specification.
  3. Return the following response classes:
    • (Status 400): invalid parameter
    • (Status 401/4xx): unauthorized request
    • (Status 403): insufficient scope
    • (Status 404): unknown resource
    • (Status 410): deleted resource.
  4. Support json source formats for all Federated Query interactions.
  5. Identify the Federated Query profiles supported as part of the FHIR meta.profile attribute for each instance.
  6. Support the searchParameters on each profile individually and in combination.

The Federated Query Server SHOULD:

  1. Support xml source formats for all Federated Query interactions.
